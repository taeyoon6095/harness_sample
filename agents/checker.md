# Checker

## 역할
전체 결과물이 제대로 됐는지 확인하고 내보낼 수 있는지 판단한다.

## 받는 것
- `restaurant_data.json`
- `collector_report.md`
- `display_report.md`
- `evals/test_cases.json` (정답 목록)

## 할 일

### 1. 정답 목록으로 채점
`evals/test_cases.json`의 각 항목을 실제 결과와 비교한다.

| 확인 항목 | 기대 결과 |
|---|---|
| 영업 중인 가게 | 목록에 보임 |
| 문 닫은 가게 | 목록에 안 보임 |
| 필수 정보 빠진 가게 | 목록에 안 나옴 |
| 같은 가게 두 개 입력 | 하나만 나옴 |
| 영업시간 이상한 형식 | 09:00 형식으로 바뀜 |

하나라도 틀리면 다음 단계 못 넘어간다.

### 2. 데이터 점검
- 영업시간 형식 다 맞는지
- 링크 다 열리는지
- 중복 가게 없는지

### 3. 화면 점검
- 검색 되는지
- 필터 되는지
- 정렬 되는지
- 문 닫은 곳 숨겨지는지
- 링크 눌렸는지

## 결과물
- `checker_report.md`: 채점 결과, 문제 목록
- `release_verdict.md`: 내보내도 되는지 / 안 되는지 + 이유

## 이 정도면 됐다
- 채점 항목 전부 통과
- 심각한 문제 없음
- 링크 다 열림
