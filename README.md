# harness_eng_with

```
AGENTS.md를 먼저 읽고 프로젝트 규칙을 따라 개발해.
작업 시 하위 역할로 분리해 진행하라.

1. Research agent
- 대한민국 AI 관련 정부 지원사업 / 공모전 데이터 소스 최소 3개 조사
- 각 소스의 구조와 수집 난이도 요약

2. Parser agent
- 표준 데이터 스키마 정의
- 최소 3개 데이터 소스 parser 구현
- 중복 제거 규칙 적용

3. Frontend agent
- 카드형 목록 UI 구현
- 검색 / 필터 / 정렬 기능 구현
- 마감 지난 항목 기본 숨김 처리

4. QA agent
- qa-deadline-links skill 기준으로 검증
- release-readiness 기준으로 최종 점검
- 링크 유효성, 날짜 파싱, 중복 여부 점검

전체 작업 순서:
1. 각 agent가 자신의 계획을 먼저 제출
2. 가능한 작업은 병렬로 수행
3. 결과를 통합
4. 충돌이나 리스크를 정리
5. README 업데이트

완료 기준:
- build 통과
- 검색/필터 동작
- 마감 지난 항목 숨김
- 중복 제거 동작
- 링크 유효성 검증 결과 포함

최종 출력:
- agent별 작업 요약
- 변경 파일 요약
- 검증 결과
- 남은 리스크
```
